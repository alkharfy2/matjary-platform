## نبذة عامة

منصة **متجري** أنهت بنجاح **27 ميزة** عبر 4 مراحل (P0-P3):
- ✅ P0: Pixels + WhatsApp + Emails + CSV Export + Invoice + Notifications (8 مميزات)
- ✅ P1: Fake Order Blocker + Abandoned Cart + Upsell + Cross-sell + Down-sell + Snapchat Pixel (6 مميزات)
- ✅ P2: AI Store Builder + AI Product Description + AI Marketing + AI Insights (4 مميزات)
- ✅ P3: Dropshipping + Affiliate + Blog + PWA + Loyalty + Multi-lang + Advanced Coupons + Shipping Accounts + Mobile App (9 مميزات)

**لكن المنافسين عندهم مميزات إضافية** تغطي فجوات مهمة في تجربة المستخدم. التحليل أظهر إن:

1. **بدون حسابات عملاء** — نظام الولاء والـ Wishlist مش كامل، والعميل بيدخل بياناته كل مرة
2. **بدون Quick Checkout** — YouCan بتسوّقها كأهم ميزة تنافسية: "خطوات أقل = مبيعات أكثر"
3. **بدون Facebook CAPI** — 95% من التجار المصريين على Facebook Ads، والـ client-side pixel مش كافي بعد iOS 14+
4. **بدون Order Tracking** — العميل بيتصل بالتاجر كل ساعة يسأل فين طلبه
5. **بدون Bundles & Countdown** — أدوات زيادة المبيعات اللي كل المنافسين عندهم

### 1. حسابات العملاء (Customer Accounts)

**الأولوية**: P4 — حرجة  
**الجهد**: كبير (3-5 أيام)  
**التأثير**: يفتح الباب لـ Wishlist + Loyalty الكامل + إعادة الطلب + عناوين محفوظة  
**المنافسون**: سلة ✅ | YouCan ✅ | Shopify ✅ | EasyOrders ✅ | Zid ✅

#### لماذا؟

بدون حسابات عملاء:
- العميل بيدخل بياناته (اسم + رقم + عنوان) **كل مرة**
- نظام الولاء مش هيشتغل بالكامل (العميل مش عارف رصيده)
- مفيش Wishlist ممكنة
- مفيش "طلباتي السابقة"
- مفيش "إعادة الطلب بضغطة واحدة"

#### الفكرة العامة:

```
العميل يزور المتجر
    ├── يضغط "تسجيل" أو "دخول"
    │   ├── تسجيل بـ: رقم الموبايل + OTP (الأسهل في مصر)
    │   ├── أو: إيميل + كلمة سر
    │   └── أو: Google OAuth (اختياري)
    │
    ├── بعد الدخول:
    │   ├── بياناته تتعبى تلقائياً في Checkout
    │   ├── يشوف "طلباتي" — كل طلباته السابقة
    │   ├── يشوف "رصيد نقاطي" — نقاط الولاء
    │   ├── يشوف "المفضلة" — منتجاته المحفوظة
    │   ├── يشوف "عناويني" — عناوين الشحن المحفوظة
    │   └── يضغط "إعادة الطلب" — يعيد طلب سابق بضغطة
    │
    └── التاجر في الداشبورد:
        ├── يشوف العملاء المسجلين
        ├── يشوف سجل مشتريات كل عميل
        └── يبعت إشعارات لعملاء محددين
```

#### جدول قاعدة البيانات:

```sql
-- جدول حسابات العملاء (مستقل عن store_customers الموجود)
store_customer_accounts
  - id: UUID (PK)
  - store_id: UUID (FK → stores, NOT NULL)
  - customer_id: UUID (FK → store_customers, UNIQUE per store, nullable)
  - phone: text (NOT NULL) — رقم الموبايل (المعرّف الأساسي)
  - phone_verified: boolean (default false)
  - email: text (nullable)
  - password_hash: text (nullable) — bcrypt hashed
  - name: text (NOT NULL)
  - avatar_url: text (nullable)
  - default_address: JSONB (ShippingAddress, nullable)
  - saved_addresses: JSONB (ShippingAddress[], default [])
  - auth_provider: text (default 'phone') — 'phone' | 'email' | 'google'
  - last_login_at: timestamp(tz)
  - is_active: boolean (default true)
  - created_at: timestamp(tz) (default now())
  - updated_at: timestamp(tz) (default now())
  
  UNIQUE(store_id, phone)
  INDEX(store_id, email)
```

```sql
-- جدول OTP مؤقت
store_customer_otps
  - id: UUID (PK)
  - store_id: UUID (FK → stores)
  - phone: text (NOT NULL)
  - otp_code: text (NOT NULL) — 6 أرقام hashed
  - expires_at: timestamp(tz) (NOT NULL) — +5 دقائق
  - attempts: integer (default 0) — حد أقصى 3 محاولات
  - is_used: boolean (default false)
  - created_at: timestamp(tz)
  
  INDEX(phone, store_id)
```

#### التكامل مع الأنظمة الموجودة:

1. **Checkout**: لو العميل مسجل → بياناته تتعبى تلقائياً (الاسم، الرقم، العنوان)
2. **Loyalty**: ربط `store_customer_accounts.customer_id` مع `store_loyalty_points.customer_id`
3. **Abandoned Cart**: لو العميل مسجل → نعرف مين سابح عربة التسوق بدون ما يدخل بياناته
4. **Reviews**: ربط التقييم بحساب العميل + شارة "مشتري مؤكد"
5. **Fake Order Blocker**: ربط الحساب بـ `platform_customers` — العميل المسجل ثقته أعلى تلقائياً

### 2. قائمة المفضلة (Wishlist)

**الأولوية**: P4 — حرجة  
**الجهد**: متوسط (1-2 يوم)  
**التأثير**: يرفع معدل الرجوع للمتجر بنسبة 25%+ ويحسّن تجربة التصفح  
**المنافسون**: سلة ✅ | YouCan ✅ | Shopify ✅ | EasyOrders ✅  
**يعتمد على**: ميزة #1 (حسابات العملاء)

#### الفكرة:

```
العميل يضغط ❤️ على أي منتج
    ├── لو مسجل → يتحفظ في قاعدة البيانات (Persistent)
    └── لو مش مسجل → يتحفظ في localStorage (مؤقت)
        └── لما يسجل → ينقل من localStorage للـ DB

العميل يروح صفحة "المفضلة":
    ├── يشوف كل المنتجات اللي حفظها
    ├── يشوف السعر الحالي (ولو السعر اتغيّر يشوف تنبيه)
    ├── يقدر يضيف للسلة مباشرة
    ├── يقدر يشيل من المفضلة
    └── يقدر يشارك المفضلة كرابط
```

#### جدول قاعدة البيانات:

```sql
store_wishlists
  - id: UUID (PK)
  - store_id: UUID (FK → stores, NOT NULL)
  - customer_account_id: UUID (FK → store_customer_accounts, NOT NULL)
  - product_id: UUID (FK → store_products, NOT NULL)
  - variant_id: text (nullable) — لو حفظ variant معين
  - price_when_added: decimal(10,2) — السعر وقت الإضافة (لتتبع تغيير السعر)
  - created_at: timestamp(tz) (default now())
  
  UNIQUE(store_id, customer_account_id, product_id, variant_id)
  INDEX(store_id, customer_account_id)
  INDEX(product_id)
```

### 3. الطلب السريع (Quick Checkout)

**الأولوية**: P4 — حرجة  
**الجهد**: متوسط (1-2 يوم)  
**التأثير**: يقلل خطوات الشراء من 4 إلى 2 — يرفع التحويل 15-25%  
**المنافسون**: YouCan ✅ (ميزة رئيسية) | EasyOrders ✅ | سلة ✅

#### الفكرة:

```
صفحة المنتج:
  ├── زر "أضف للسلة" ← الأساسي (يضيف ✕ يكمل)
  │
  └── زر "🛒 اشتري الآن" ← Quick Checkout
      │
      ├── العميل مسجل + عنده عنوان → ينتقل لـ Checkout مع البيانات جاهزة
      │
      └── العميل مش مسجل → ينتقل لـ Checkout مع المنتج فقط في السلة
          └── يملا بياناته مرة واحدة → يدفع
```

```typescript
/**
 * Facebook Conversion API (Server-side Events)
 * https://developers.facebook.com/docs/marketing-api/conversions-api
 */

interface CAPIEvent {
  event_name: 'Purchase' | 'InitiateCheckout' | 'AddToCart' | 'ViewContent' | 'Lead'
  event_time: number  // Unix timestamp
  event_id: string    // للـ deduplication مع Client Pixel
  event_source_url: string
  user_data: {
    em?: string[]    // SHA-256 hashed emails
    ph?: string[]    // SHA-256 hashed phones (format: 201234567890)
    fn?: string[]    // SHA-256 hashed first names
    ct?: string[]    // SHA-256 hashed city
    country?: string[] // SHA-256 hashed country code
    client_ip_address?: string
    client_user_agent?: string
    fbc?: string     // Facebook click ID (from _fbc cookie)
    fbp?: string     // Facebook browser ID (from _fbp cookie)
  }
  custom_data?: {
    value?: number
    currency?: string
    content_ids?: string[]
    content_type?: string
    contents?: { id: string; quantity: number; item_price: number }[]
    order_id?: string
    num_items?: number
  }
  action_source: 'website'
}

async function sendConversionEvent(
  pixelId: string,
  accessToken: string,
  events: CAPIEvent[],
  testEventCode?: string
): Promise<void> {
  const url = `https://graph.facebook.com/v19.0/${pixelId}/events`

  const body: Record<string, unknown> = {
    data: events,
    access_token: accessToken,
  }

  if (testEventCode) {
    body.test_event_code = testEventCode
  }

  const response = await fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(body),
  })

  if (!response.ok) {
    console.error('[CAPI] Error:', await response.text())
  }
}
```


```
   → يرجع: رقم التتبع + بوليصة PDF
   → يتحدث tracking_number في الطلب
3. شركة الشحن تحدّث الحالة (webhook أو polling)
   → حالة الطلب تتحدث تلقائياً
4. العميل يتتبع طلبه من صفحة التتبع
```

```sql
  - order_id: UUID (FK → store_orders, UNIQUE)
  - shipping_account_id: UUID (FK → store_shipping_accounts)
  - provider: text (NOT NULL) — 'bosta' | 'aramex' | 'jt' | 'mylerz' | 'sprint'
  - external_id: text — ID in provider system
  - tracking_number: text
```

## المرحلة P4-A — حسابات العملاء وتجربة الشراء

> **الأساس اللي كل حاجة تانية هتبني عليه**  
> **الجهد**: ~5-7 أيام | **التأثير**: ★★★★★

### لماذا مع بعض؟

```
حسابات العملاء (#1) ←──── أساس ────→ المفضلة (#2) تعتمد عليه 100%
        │
        └── الطلب السريع (#3) ← لو العميل مسجل + عنده عنوان = شراء بضغطة واحدة
```

- **بدون حسابات عملاء** → المفضلة مش هتشتغل persistent
- **بدون حسابات عملاء** → Quick Checkout مش هيقدر يعبّي البيانات تلقائياً
- الثلاثة مع بعض بيكمّلوا **قصة مستخدم واحدة**: "أنا عميل → أسجل → أحفظ منتجات → أشتري بسرعة"

## المرحلة P4-B — التتبع والثقة والمراجعات

> **ثقة العميل + دقة بيانات التاجر = مبيعات أكتر**  
> **الجهد**: ~4-5 أيام | **التأثير**: ★★★★★

### لماذا مع بعض؟

```
Facebook CAPI (#4) → التاجر يتتبع مبيعاته بدقة → إعلانات أفضل
        +
المراجعات (#5) → العميل يثق في المنتج → يشتري → الإعلان ينجح
        +
تتبع الطلب (#8) → العميل مرتاح → يقيّم إيجابي → المراجعات تتحسن
```

- **CAPI** بيرفع جودة بيانات التاجر (ROAS أفضل)
- **المراجعات** بترفع التحويل (العميل يثق)
- **تتبع الطلب** بيقلل مشاكل "فين طلبي" + يخلي العميل يقيّم بعد التوصيل
- الثلاثة بيخدموا دورة واحدة: **إعلان → شراء → تتبع → تقييم → إعلان أفضل**

```typescript
// facebook-capi.ts
async function sendConversionEvent(
  pixelId: string,
  accessToken: string,
  events: CAPIEvent[],
  testEventCode?: string
): Promise<void> {
  const url = `https://graph.facebook.com/v19.0/${pixelId}/events`
  await fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      data: events,
      access_token: accessToken,
      ...(testEventCode && { test_event_code: testEventCode }),
    }),
  })
}
```

#### Deduplication:

// Client-side:

// Server-side:
